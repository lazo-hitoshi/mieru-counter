## 1. 全体の流れ

```
① スタッフがセッションを作成
     ↓
② 接続コードを利用者に伝える
     ↓
③ 利用者がコードを入力して参加
     ↓
④ スタッフが話す/入力 → 利用者の画面にリアルタイム字幕表示
     ↓
⑤ 重要事項を利用者に送信 → カードとして表示
     ↓
⑥ 利用者が確認ボタンで反応 → スタッフ画面に通知
     ↓
⑦ 対応完了後、スタッフがセッションを終了
```

## 3. スタッフ画面

**URL:** `http://localhost:3000/staff`

窓口スタッフが使用する画面です。セッション管理、メッセージ送信、重要事項カードの管理を行います。

### 3.1 画面構成

```
┌──────────┬───────────────────────┬──────────┐
│          │                       │          │
│ セッション │    会話エリア          │ 重要事項  │
│ 一覧      │   （チャット形式）      │ カード    │
│          │                       │          │
│          ├───────────────────────┤          │
│          │ 📋 🎙️ [入力欄] [送信]  │          │
└──────────┴───────────────────────┴──────────┘
```

### 3.2 セッションの作成

1. 左サイドバーの **「＋新しいセッション」** ボタンをクリック
2. セッションが作成され、**接続コード**（例: `A1B2C3D4`）が発行されます
3. この接続コードを利用者に伝えてください（口頭、メモ、QRコードなど）

### 3.3 セッションの選択

- 左サイドバーに作成済みセッションが一覧表示されます
- クリックすると、そのセッションの会話エリアが表示されます
- 各セッションにはステータスが表示されます：
  - **待機中**（黄色）：利用者が未接続
  - **対応中**（緑色）：利用者が接続済み
  - **終了**（灰色）：セッション終了済み

### 3.4 メッセージの送信

#### キーボード入力
1. 画面下部の入力欄にメッセージを入力
2. **Enter キー** または **「送信」ボタン** をクリック
3. メッセージが利用者画面にリアルタイムで字幕として表示されます

#### 音声入力（マイクボタン）
1. 入力欄の左にある **🎙️（マイクボタン）** をクリック
2. マイクが赤く光り、音声認識が開始されます
3. 通常通り話してください
4. 認識されたテキストが自動的に入力欄に追加されます
5. 内容を確認し、必要に応じて修正してから **「送信」** をクリック
6. もう一度マイクボタンをクリックすると、音声認識が停止します

> **注意:** 音声入力は Google Chrome で利用できます。Firefox では利用できません。

#### 定型文の使用
1. 入力欄の左にある **📋（定型文ボタン）** をクリック
2. カテゴリ別（受付・服薬・会計・案内）に定型文が表示されます
3. 使いたい定型文をクリックすると、すぐに送信されます

### 3.5 重要事項カードの管理

会話内容からAIが自動的に重要な情報（薬の飲み方、次回予約、金額など）を抽出し、右サイドバーに**候補カード**として表示します。

#### カードの状態
| 状態 | 色 | 説明 |
|------|-----|------|
| AI候補 | 黄色 | AIが抽出した候補。スタッフの確認待ち |
| 確認済み | 青色 | スタッフが内容を確認済み |
| 送信済み | 緑色 | 利用者の画面に送信済み |

#### 操作手順
1. AI候補のカードが表示されたら、内容を確認します
2. 正しい場合は **「確認」** ボタンをクリック
3. 不要な場合は **「却下」** ボタンをクリック
4. 確認済みのカードを利用者に送るには **「利用者へ送信」** ボタンをクリック

### 3.6 呼び出し機能

待合室にいる利用者を呼び出す機能です。

1. ヘッダーの **「📢 呼び出し」** ボタンをクリック
2. 呼び出し番号を入力（例: `15`）
3. **「送信」** をクリック
4. 利用者の画面に全画面で呼び出し番号が表示されます

### 3.7 利用者の反応の確認

利用者が確認ボタンを押すと、会話エリアの下に反応が表示されます。

- **緑色**：「理解しました」
- **オレンジ色**：「もう一度お願いします」「ゆっくりお願いします」
- **紫色**：「文字でください」「手話通訳が必要です」

新しい反応があると、ヘッダーに **「利用者から反応あり」** と赤いバッジが表示されます。

### 3.8 セッションの終了

1. ヘッダーの **「終了」** ボタンをクリック
2. 確認ダイアログが表示されます
3. **「OK」** をクリックするとセッションが終了します
4. 利用者の画面にも「セッション終了」と表示されます

## 4. 利用者画面

**URL:** `http://localhost:3000/visitor`

来訪者（聴覚障害者・難聴者・高齢者）が使用する画面です。スマートフォンでの利用を想定しています。

### 4.1 セッションへの参加

1. 利用者画面を開きます
2. スタッフから受け取った**接続コード**を入力します
3. **「参加する」** ボタンをタップ
4. 接続に成功すると、字幕表示画面に切り替わります

### 4.2 画面構成

```
┌─────────────────────┐
│ 施設名   [A+][A-][メモ]│  ← ヘッダー
├─────────────────────┤
│                     │
│  スタッフのメッセージ   │  ← 字幕エリア（大きな文字）
│  がここに表示されます   │
│                     │
├─────────────────────┤
│  📌 重要事項カード     │  ← 送信された重要事項
├─────────────────────┤
│ [理解] [もう一度] [ゆっくり]│
│ [文字でください] [手話通訳] │  ← 確認ボタン
└─────────────────────┘
```

### 4.3 字幕の表示

- スタッフが送信したメッセージが**大きな文字**でリアルタイム表示されます
- 最新のメッセージが常に表示されます

### 4.4 文字サイズの変更

ヘッダー右上の **「A+」** と **「A-」** ボタンで文字サイズを調整できます。

- **A+**：文字を大きくする（最大 36px）
- **A-**：文字を小さくする（最小 14px）

### 4.5 確認ボタン

画面下部に5つの確認ボタンがあります。タップするとスタッフの画面に反応が通知されます。

| ボタン | 意味 | 使う場面 |
|--------|------|---------|
| ✓ 理解しました | 内容を理解した | 説明を理解できたとき |
| 🔄 もう一度 | もう一度説明してほしい | 聞き取れなかったとき |
| 🐢 ゆっくり | ゆっくり話してほしい | 話が速いとき |
| ✏️ 文字で | 文字で書いてほしい | 口頭では理解しにくいとき |
| 🤟 手話通訳 | 手話通訳が必要 | 手話での対応を希望するとき |

ボタンを押すと **「✓ スタッフに伝えました」** と表示されます。

### 4.6 重要事項カードの確認

スタッフから送信された重要事項が、字幕エリアの下にカードとして表示されます。

- **赤枠**：緊急の事項
- **黄枠**：重要な事項
- **青枠**：通常の情報

それぞれのカードには、種類を示すアイコン（💊薬、📅予約、💰支払いなど）が表示されます。

### 4.7 会話メモ

ヘッダー右の **「会話メモ」** ボタンをタップすると、以下が一覧で確認できます：

- 送信された重要事項のまとめ
- 会話履歴の全文

**「字幕に戻る」** ボタンで通常の字幕表示に戻ります。

### 4.8 呼び出し通知

スタッフが呼び出しを行うと、**全画面でオレンジ色の呼び出し画面**が表示されます。

- 呼び出し番号が大きく表示されます
- **「閉じる」** ボタンで通常画面に戻ります
- 15秒後に自動で閉じます

## 5. Lens Preview（スマートグラス）

**URL:** `http://localhost:3000/lens`

Even G2 スマートグラスでの表示をシミュレートする画面です。

### 5.1 接続

1. Lens Preview 画面を開きます
2. **接続コード**を入力します
3. **「接続する」** ボタンをクリック

### 5.2 表示モード

スマートグラスのビューポートに、3種類の情報が自動で切り替わって表示されます：

| モード | 表示内容 | インジケーター |
|--------|---------|-------------|
| 字幕 | スタッフのメッセージ | 左のドット点灯 |
| 重要事項 | 送信された重要事項カード | 中央のドット点灯 |
| 呼び出し | 呼び出し番号 | 右のドット点灯 |

### 5.3 履歴バー

画面下部に最近の表示履歴が小さなボタンとして並びます。クリックすると、過去の表示内容を再確認できます。

## 6. 管理画面

**URL:** `http://localhost:3000/admin`

施設の管理者が使用する画面です。3つのタブがあります。

### 6.1 施設管理タブ

登録されている窓口の一覧を確認できます。

- 窓口名
- 窓口コード
- QRトークン（利用者の接続用）
- ステータス（有効/無効）

### 6.2 セッション履歴タブ

過去のセッション一覧を表形式で確認できます。

| 列 | 説明 |
|----|------|
| コード | セッションの接続コード |
| 窓口 | 使用された窓口名 |
| 状態 | 対応中/待機中/終了 |
| 利用者 | 利用者のラベル |
| 開始 | セッション開始日時 |

### 6.3 定型文管理タブ

登録されている定型文の一覧を確認できます。

- カテゴリ（受付・服薬・会計・案内）
- タイトル
- 本文
- ステータス（有効/無効）

## 7. よくある質問

### Q. 音声入力のマイクボタンが表示されません

**A.** マイクボタンは以下の条件で表示されます：
- **Google Chrome** を使用していること（Firefox は非対応）
- セッションを選択して入力エリアが表示されていること
- ブラウザがマイク使用を許可していること（初回使用時にブラウザから許可を求められます）

### Q. 利用者の画面にメッセージが表示されません

**A.** 以下を確認してください：
- 利用者が正しい接続コードで参加しているか
- セッションのステータスが「対応中」になっているか
- ブラウザのネットワーク接続が安定しているか

### Q. 接続コードはどこに表示されますか？

**A.** スタッフ画面でセッションを選択すると、ヘッダー部分に「利用者接続コード」として表示されます。

### Q. 1つのセッションに複数の利用者が接続できますか？

**A.** 同じ接続コードで複数の端末から接続可能です。スマートフォンとスマートグラスの両方を同時に使うこともできます。

### Q. セッション終了後にデータは残りますか？

**A.** セッション履歴、会話ログ、重要事項カードは管理画面から確認できます。
